# v0.7.6 - 2026-06-10

Security + performance audit batch (June 2026) and mobile UX fixes. All changes
verified live on the iOS simulator; `tsc` and `cargo check` clean.

## Security
- **Deepgram key removed from the build.** `build.rs` no longer embeds
  `DEEPGRAM_API_KEY`; `transcribe_deepgram` requires a user-provided key
  (new password field in Settings → STT, shown for the Deepgram provider,
  persisted via `sttStore`). The previously embedded key was revoked.
- **Path validation in `transcribe_audio`**: the input file is canonicalized
  and must live under the app data dir — IPC can no longer point the
  transcriber at arbitrary files.
- **URL scheme allowlist in `MarkdownRenderer`**: links/images render only
  `http/https/tg/mailto` URLs (control characters stripped before the check);
  anything else falls back to plain text. Blocks `javascript:`/`data:`/`file:`.
- **`tauri.conf.json` hygiene**: `devtools` removed, `withGlobalTauri: false`,
  `assetProtocol.scope` narrowed from `**` to `$APPDATA/media/**` (all
  `convertFileSrc` callers resolve under `app_data_dir/media`).
- **PII**: phone numbers masked in login logs (last 4 digits kept).
- Removed stale `auth.rs.bak`, `LoginForm.tsx.bak`, `fix_auth.patch`.

## Performance / Reactivity
- **Virtualized ChatList** (`@tanstack/react-virtual`): only visible rows in
  the DOM, per-row `measureElement`, density-aware estimates, keyboard
  highlight follows via `scrollToIndex`.
- **Virtualized MessageList**: bottom-anchored via `scrollToIndex(last, 'end')`;
  history prepend keeps the scroll anchored to the previously-oldest message;
  jump-to-message, pending-scroll and drag-select preserved. Message grouping
  memo takes primitives, so a new message no longer re-renders the whole feed.
- **Live chat list** (`telegram:new-message` listener in `MainLayout`):
  updates `lastMessage` preview, bumps unread (skips the open chat and
  outgoing), moves the chat to the top — account-filtered.
- **Re-render hotspots**: rAF-throttled `telegram:call-audio-level`,
  rAF-batched `chat-avatar-updated`, proper Zustand selectors
  (mute state / account switcher / voice transcription keyed by message),
  `t()` memoized per language, stable keydown + chat-click handlers,
  `whisper-progress` listener mounted only while transcribing.
- Autoscroll on new messages only when within ~150 px of the bottom; the
  message textarea stays enabled while sending.

## Mobile (iOS/Android)
- **No document overscroll**: the whole app could be dragged down past the
  top (WKWebView rubber-band). Fixed in CSS (`overscroll-behavior: none`,
  `position: fixed` body) and natively — `main.mm` swizzles
  `WKWebView initWithFrame:configuration:` to set
  `scrollView.bounces/alwaysBounce*/bouncesZoom = NO`.
- **No UI zoom**: viewport meta `maximum-scale=1, user-scalable=no`
  (WKWebView honors it; Android WebView too) + `gesturestart/change/end`
  `preventDefault` + `touch-action: manipulation`. Double-tap zoom inside
  `ImageViewer`/`AvatarViewer` is intentionally kept.
- **Viewport-scoped media downloads**: `MediaAttachment` auto-downloads
  (photo/sticker/voice) only when the placeholder is actually on screen
  (IntersectionObserver, 100 px margin, 150 ms debounce against flings);
  leaving the viewport cancels still-queued downloads; an explicit
  tap-to-download (video/audio/document) jumps to the front of the queue
  instead of waiting behind auto-downloads.
  Measured in a media-heavy channel: opening = 4 files, 3 screens of
  scrolling = +3, four fast flings = +9 (previously: the entire loaded
  history batch was queued at once).
