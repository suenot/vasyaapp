## Security

### D3 ‚Äî encrypted sessions at rest
- New `telegram/encrypted_session.rs`: an in-memory `grammers` `Session`
  implementation that persists an encrypted snapshot
  (ChaCha20-Poly1305) to `<account>.session.enc`.
- Master key from the OS keychain via the `keyring` crate
  (`apple-native` / `windows-native`); fallback to a `0600` `.session.key`
  file when no keychain is available.
- Write strategy: auth keys / home-DC / self-peer flush **eagerly**; the
  high-frequency `updates_state` is throttled (‚â§ every 2 s) and flushed on
  app `Exit` (`RunEvent::Exit` ‚Üí `flush_all_sessions`). Losing the last
  seconds of `pts/qts` on a crash is safe (grammers catches up); losing an
  auth key is not ‚Äî hence the split.
- `peer(PeerId::self_user())` resolves to the cached `is_self` user (grammers
  uses it to detect "already logged in" ‚Äî `MemorySession` does not do this).
- **Migration**: a legacy plaintext `SqliteSession` is imported via
  `SessionData::from` (auth keys + self peer + updates state preserved), the
  encrypted snapshot is written, and only then is the plaintext file deleted.
- Verified on the simulator: `.session` ‚Üí `.session.enc`, key landed in the
  Keychain (no key-file), and the account stayed logged in after migration.

### D5 ‚Äî per-user account isolation on the sync backend
- `auth::ensure_account_access`: claims an unowned account for the
  authenticated user on first touch, then enforces `user_accounts` membership;
  used by every chats/folders/tabs handler.
- `AuthPolicy { require_user_auth }` (env `REQUIRE_USER_AUTH`, default off):
  when on, the legacy shared API key is rejected on data routes (JWT only).
- Client: email/password sign-in & registration in Settings ‚Üí Storage; the
  returned JWT is placed in the Bearer field. Token TTL defaults to 30 days
  (`JWT_TTL_HOURS`) since it doubles as the long-lived sync credential.
- Bug found & fixed along the way: axum 0.7 path params use `/:id`, not
  `/{id}` ‚Äî `delete_folder` had been silently 404-ing.

### D9 ‚Äî rate limiting + JWT_SECRET fail-fast
- `rate_limit.rs`: dependency-free in-memory fixed-window limiter (10 req/min
  per IP) on `/api/auth/login` and `/api/auth/register`; server uses
  `into_make_service_with_connect_info` to expose the client IP.
- `JWT_SECRET` panics on startup if explicitly set but `< 32` chars.

### D6 / D7 / D8
- D6: `SttProvider::default()` ‚Üí `LocalWhisper` (Rust + `sttStore`). Privacy by
  default ‚Äî audio stays on device unless the user opts into Deepgram.
- D7: calls hidden behind `experimentalCalls` (off). ChatHeader call buttons
  hidden; `callStore` ignores incoming calls; Settings toggle warns the audio
  is not E2E-encrypted yet.
- D8: CSP tightened ‚Äî `object-src 'none'`, `frame-src 'none'`,
  `base-uri 'self'`, `form-action 'self'`, explicit `font-src`/`media-src`
  (Google Fonts). `connect-src` stays `https:` (arbitrary self-hosted backend).

## Features
- **My QR Code** (`components/Profile/MyQrCode.tsx`): a `t.me/<username>` QR
  like the native app, opened from the sidebar header. Falls back to a
  `+phone` link with a privacy hint when no username is set; "no username"
  guidance otherwise. Uses `qrcode.react`. Verified live (`Evgeniy @suenot`).

## Performance / refactor
- **U-file**: `send_media` now takes a `tauri::ipc::Request` ‚Äî the file bytes
  are the RAW IPC body (`InvokeBody::Raw`) and metadata travels in
  percent-encoded `x-*` headers. Previously every byte was serialized as a
  decimal number in a JSON array (~4√ó payload + parse cost). Frontend:
  `invoke('send_media', new Uint8Array(buffer), { headers })`.
  (Live file-send QA still pending a rebuild.)
